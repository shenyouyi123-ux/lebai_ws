# 机械臂巡检点操作

## 项目背景

本文档基于当前工作区实际联调过程整理，目标是提供一份可直接执行的巡检点配置与巡检启动手册。

环境与前提：
- ROS 2 Humble + MoveIt 2
- Ubuntu 22.04
- 工作空间：`~/lebai_ws`
- 机械臂：乐白 LM3
- 巡检包：`lebai_inspection_system`（新建，零侵入复用原 MoveIt 包）

---

## 本次联调结论（关键）

1. 巡检功能通过独立包实现，不改原有真实/仿真 MoveIt 主流程。
2. 仿真不动的根因曾出现两类问题：
   - URDF 中 `FakeSystem` 重复定义，导致 `ros2_control_node` 崩溃。
   - `moveit_controllers.yaml` 命名空间不符合 `MoveItSimpleControllerManager` 预期。
3. 已修复后，仿真可正常加载控制器并按巡检点自动执行。
4. RViz 手动示教时，若自动巡检同时运行，容易出现 `Plan and Execute request aborted`（控制器并发抢占）。

## 巡检点配置说明

`inspection_points.yaml` 中每个点位包含：

```yaml
inspection_points:
  - name: p01_xxx
    joints: [j1, j2, j3, j4, j5, j6]  # 单位：rad
    gripper_rad: 0.0                   # 0.0 关闭，1.0 打开（按当前桥接逻辑）
```

规则：
1. `joints` 必须始终是 6 个值（对应 `joint_1` 到 `joint_6`）。
2. 推荐使用弧度，转换公式：

$$
\text{rad} = \text{deg} \times \frac{\pi}{180}
$$

3. 先仿真验证可达，再上真实机。

## 启动流程

### 1. 编译与环境加载

```bash
cd ~/lebai_ws
colcon build --packages-select \
  lebai_with_gripper_and_shaft_moveit_config \
  lebai_gripper_bridge \
  lebai_inspection_system
source install/setup.bash
```

### 2. 仿真巡检（自动开始）

```bash
ros2 launch lebai_inspection_system inspection_sim.launch.py auto_start:=true
```

### 3. 仿真示教模式（不自动巡检，推荐调点时使用）

```bash
ros2 launch lebai_inspection_system inspection_sim.launch.py auto_start:=false
```

### 4. 真实机械臂巡检

```bash
ros2 launch lebai_inspection_system inspection_real.launch.py robot_ip:=10.20.17.1 auto_start:=true
```

### 5. 真实机先示教再执行

```bash
ros2 launch lebai_inspection_system inspection_real.launch.py robot_ip:=10.20.17.1 auto_start:=false
```

## RViz 示教巡检点（推荐流程）

1. 使用 `auto_start:=false` 启动，避免任务节点抢占控制器。
2. `Planning Group` 选择 `manipulator`。
3. 先 `Plan`，成功后再 `Plan and Execute`。
4. 每次只小幅调整一个方向（优先先平移后旋转）。
5. 贴近工件前先经过“接近位”（approach pose），不要一步到位。
6. 记录当前 6 轴关节值并写入 `inspection_points.yaml`。

## 常见问题与处理

### 1) `Could not contact service /controller_manager/list_controllers`

原因：`ros2_control_node` 未正常运行（历史上由 `FakeSystem` 重复定义引发）。

处理：
1. 确保修复后的配置已编译并 `source`。
2. 清理旧进程后重启：

```bash
pkill -f ros2_control_node
pkill -f move_group
pkill -f rviz2
pkill -f task_orchestrator
```

### 2) `Plan and Execute request aborted` / `move() failed or timeout reached`

常见原因：
1. 自动巡检与 RViz 手动执行并发冲突。
2. 目标姿态贴近障碍物或接近奇异位形。

处理建议：
1. 示教时使用 `auto_start:=false`。
2. 先走“中间接近点”，再走检查点。
3. 缩小单步姿态变化，先验证可达后再串点。

### 3) 进程超时 `Exit Code 124`

说明：多数是使用了 `timeout xxs` 导致命令被主动截断，不等同于功能失败。

## 重复巡检说明

当前实现默认执行一轮（按点位顺序跑完后结束）。

可选做法：
1. 每轮结束后手动重启 launch。
2. 后续可增加 `repeat:=true` 参数实现自动循环（当前版本未默认开启）。

---

## 真实机安全建议

1. 启动自动巡检前，先让机械臂靠近第一个点位（p01）。
2. 首次联调只保留少量点位（例如 1~4）。
3. 每次改点后先仿真跑通，再切真实机。
4. 出现异常立即停止并回到安全位重新示教。
