# lebai_vision_grasp 操作说明

Lebai LM3 机械臂 + 平行夹爪 + MoveIt 2 视觉抓取包。支持仿真抓取和真机视觉抓取。

## 目录结构

```text
lebai_vision_grasp/
├── config/
│   ├── vision_grasp_sim_params.yaml   # 仿真参数
│   └── vision_grasp_params.yaml       # 真机参数
├── launch/
│   ├── vision_grasp_sim.launch.py     # 仿真启动
│   └── vision_grasp_real.launch.py    # 真机启动
└── lebai_vision_grasp/
    ├── vision_grasp_orchestrator.py   # 抓取主控节点
    ├── fake_target_position.py        # 仿真目标发布节点
    └── gripper_sim_node.py            # 仿真夹爪节点
```

## 编译

```bash
cd ~/lebai_ws
colcon build --packages-select lebai_vision_grasp --symlink-install
source install/setup.bash
```

修改 Python 代码、launch 文件或参数文件后，建议重新编译并重启相关 launch。

## 仿真流程

仿真参数文件：

```text
~/lebai_ws/src/lebai_vision_grasp/config/vision_grasp_sim_params.yaml
```

### 一键启动

```bash
cd ~/lebai_ws
source install/setup.bash
ros2 launch lebai_vision_grasp vision_grasp_sim.launch.py
```

启动内容：

- MoveIt + RViz
- 假目标发布节点
- 抓取编排节点
- 仿真夹爪节点

### 触发仿真抓取

```bash
cd ~/lebai_ws
source install/setup.bash
ros2 service call /vision_grasp_orchestrator/execute_once std_srvs/srv/Trigger {}
```

### 仿真目标位置

在 `vision_grasp_sim_params.yaml` 中修改：

```yaml
fixed_x_m: 0.35
fixed_y_m: 0.00
fixed_z_m: 0.11
motion_mode: fixed
```

`fixed_z_m` 通常应为桌面顶面高度 + 物体半高。

## 真机流程

真机参数文件：

```text
~/lebai_ws/src/lebai_vision_grasp/config/vision_grasp_params.yaml
```

当前真机抓取姿态：

```text
rx = 180 deg
ry = 0 deg
rz = 0 deg
```

对应 ROS 四元数：

```text
x = 1.0
y = 0.0
z = 0.0
w = 0.0
```

含义：

- `tool0 +Z` 竖直向下
- 两指连线优先沿 `base_link +X`
- 如果该姿态不可达，程序会自动尝试其它仍保持竖直向下的等价姿态

当前 30mm 方块已调通的关键参数：

```yaml
grasp_target_offset_z_m: 0.003
grasp_offset_z_m: 0.162
gripper_closed_rad: 0.0
target_object_size_x_m: 0.03
target_object_size_y_m: 0.03
target_object_size_z_m: 0.03
approach_height_m: 0.04
lift_height_m: 0.10
min_grasp_center_z_m: 0.05
table_enabled: false
```

### 终端 1：启动机械臂、MoveIt、RViz、抓取编排节点

```bash

  robot_ip:=10.20.17.1 \
```

- 乐白机械臂驱动
- 夹爪桥接节点

- RViz
- `vision_grasp_orchestrator`

### 终端 4：检查视觉目标

```bash
source /opt/ros/humble/setup.bash
source ~/lebai_ws/install/setup.bash
source ~/gemini_depth_ws/install/setup.bash
ros2 topic hz /target_position
```

查看一次目标数值：

```bash
ros2 topic echo /target_position --once
```

确认：

- `header.frame_id` 是 `base_link`
- 单位是 mm
- 方块静止时 `/target_position` 稳定
- 目标点在机械臂可达范围内

```bash

```

第一次测试只触发一次，观察完整流程：

```text
张开夹爪 -> 到预抓取点 -> 垂直下降 -> 闭爪 -> 垂直抬升
```

## 更换抓取物体

更换物体时，主要修改 `config/vision_grasp_params.yaml`。建议每次只改一类参数，然后单次触发验证。

### 物体尺寸

修改物体外接盒尺寸，单位米：

```yaml
target_object_size_x_m: 0.03
target_object_size_y_m: 0.03
target_object_size_z_m: 0.03
```

这些参数主要用于 RViz marker 和抓取后附着显示，不直接决定夹爪闭合力度。

### 视觉点到夹持点的补偿

视觉节点发布 `/target_position`，编排器实际使用：

```text
grasp_target = target_position + grasp_target_offset
```

需要修改：

```yaml
grasp_target_offset_x_m: 0.0
grasp_target_offset_y_m: 0.0
grasp_target_offset_z_m: 0.003
```

常见情况：

```text
视觉点已经是物体夹持中心：
grasp_target_offset_z_m = 0.0

视觉点是物体上表面，想夹中部：
grasp_target_offset_z_m = -物体高度 / 2

夹取位置偏左/偏右：
微调 grasp_target_offset_x_m 或 grasp_target_offset_y_m
```

现场微调建议：

- 夹取点偏高：减小 `grasp_target_offset_z_m`
- 夹取点偏低或扫桌：增大 `grasp_target_offset_z_m`
- 每次改 2-5mm，例如 `0.003 -> 0.000 -> -0.005`

### tool0 到夹持中心的偏移

```yaml
grasp_offset_z_m: 0.162
```

这是 `tool0` 到夹持中心的物理距离估计。通常换同一个夹爪时不需要频繁改；如果换夹爪、换 TCP、或发现所有物体的高度都系统性偏差，再重新标定。

### 夹爪闭合量

```yaml
gripper_open_rad: 1.0
gripper_closed_rad: 0.0
```

经验：

```text
数值越小，夹得越紧
0.0 = 完全闭合
```

调试顺序：

```text
0.4 -> 0.3 -> 0.2 -> 0.1 -> 0.0
```

如果物体柔软、易变形、易弹开，不要直接完全闭合；先从较大的值试。

### 安全高度

```yaml
approach_height_m: 0.04
lift_height_m: 0.10
min_grasp_z_m: 0.05
min_grasp_center_z_m: 0.05
min_approach_z_m: 0.09
min_lift_z_m: 0.15
```

含义：

- `approach_height_m`：预抓取点在抓取点上方的高度
- `lift_height_m`：闭爪后抬升高度
- `min_grasp_z_m`：`tool0` 最低高度保护
- `min_grasp_center_z_m`：夹持中心/指尖附近最低高度保护

不要只看 `tool0` 高度。真正容易碰桌的是指尖或夹持中心。

### 物体位置与可达性

如果出现 IK 失败或规划失败，优先调整物体位置：

```text
x 尽量放在 0.25-0.40m 范围内
y 不要偏离中心太多
z 不要过高
```

如果物体更高，先提高 `approach_height_m` 或 `min_lift_z_m`，但过高可能导致 LM3 竖直向下姿态不可达。

## 诊断日志

抓取节点会输出关键位姿：

```text
开始抓取姿态尝试...
after_approach...
after_descend...
after_gripper_close...
after_lift...
```

重点看：

```text
tool0=(x, y, z)
tool_z=(x, y, z)
predicted_center_z
```

判断方法：

- `after_descend` 已经很低：目标 Z 或补偿过低
- `after_descend` 正常，但 `after_gripper_close` 的 `tool0_z` 变低：闭爪导致末端/机械臂下沉
- `tool0_z` 没变但仍碰桌：夹爪机构闭合时扫桌，需要提高抓取点或减小闭合量
- lift 失败且日志里有 `ArmDisable`：通常是前面已经触发急停，lift 只是后续执行失败

## 注意事项

- 真机第一次测试必须手放急停旁边。
- 每次只触发一次抓取，确认安全后再重复测试。
- 每次只改一个变量，避免无法判断原因。
- 修改参数后要重新编译并重启终端 1 的 launch。
- `/target_position` 必须是 `base_link` 坐标系，单位 mm。
- `table_enabled` 当前为 `false`。如果重新启用桌面碰撞体，需要先在 RViz/实际安装中校准桌面位置，否则可能导致 MoveIt 认为机械臂起始状态碰撞。
- MoveIt 中的规划 link 是 `tool0`，不是指尖；夹爪指尖相对 `tool0` 有物理偏移。
- 当前抓取策略是竖直向下抓取。程序会自动尝试多个竖直向下的 wrist yaw 姿态，提高 IK 成功率。
- 相机 TF 警告如 `camera_color_frame` 到 `world` 不连通，一般不影响本抓取流程，只要 `/target_position` 已经稳定发布到 `base_link`。
- 如果夹爪闭合时触桌，优先提高 `grasp_target_offset_z_m` 或减小闭合量，不要继续下探。
